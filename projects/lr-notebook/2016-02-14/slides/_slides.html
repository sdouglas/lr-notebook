## Background Continuing to work on new lektor-based notebook system for the lab!!!! I've used the following for notebooks: - **per-group mediawiki instance, per-user accounts** - pros: everyone's work in one place - cons: hard to enforce organization, lacks portability, scales poorly over time - **per-user mediawiki instance** - pros: more similar to individual paper notebooks, more portable - cons: poor group-wide indexing, search. collaboration is hard. - **per-project git repo, with per-user branches** - pros: benefits of git version control: diffs, lots of git(hub) integrations available - cons: any group-wide indexing requires constent, burdensome merging Next stage in notebook evolution is: - **per-user git repo using lektor (static+cms+framework), plus group-wide indexing** - pros: combined benefits of per-user notebooks with automatic per-project indexing, and git-based version control - cons: tbd; requires some work to set up I'm currently favoring the following folder layout and repo naming convention ``` notebooks/ all/ nb-docs/ nb-han/ nb-pablo/ nb-parsa/ nb-mia/ nb-shawn/ nb-suraj/ nb-tural/ ``` - `nb-*` is an individual git repo containing a lektor notebook - `all` is a customized lektor notebook (perhaps with some scripts or symlinks) that builds a master index of sibling notebooks. - `nb-docs` will store general lab information (currently on the wiki) ## Clean up prototype lektor notebook I originally structured the root of this notebook to be a sub-folder of a git repo.
